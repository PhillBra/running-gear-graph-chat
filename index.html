Knotentypen

Der Graph modelliert die gesamte Wertschöpfungskette rund um Trailer-Achsen und Fahrwerkskomponenten.

Kantentypen

Beziehungen zwischen den Knoten — von Kompatibilität über Herstellerzuordnung bis zu Normverletzungen.

Baugruppen

Die Baugruppen des Fahrwerks mit ihren Unterbaugruppen.

Ontologie & Strukturierung

Der Graph basiert auf einer Ontologie, die sich an folgenden ISO-Normen orientiert: ISO 11452 (Fahrzeugkomponenten), ISO 7638 (Verbindungen Zugfahrzeug/Anhänger), ISO 1726 (Begriffe Anhängerfahrzeuge), ISO 11992 (Datenkommunikation Zugfahrzeug/Anhänger) und ISO 8855 (Fahrdynamik-Begriffe). Die Knoten- und Kantentypen bilden das BPW-Datenmodell mit 12 Knotentypen und 24 Kantentypen ab.

Datenquellen

Der Graph wurde aus 52 Ersatzteilkatalogen von über 30 Herstellern aufgebaut. Darunter BPW, SAF-Holland, Knorr-Bremse, WABCO, Hendrickson, Krone, Kögel, DAF, MAN, Volvo, Scania, MEYLE, JOST, Continental und viele mehr. Jeder Katalog wurde maschinell geparst — Teilenummern, Cross-References und Bauteil-Metadaten extrahiert.

Zusätzlich enthält der Graph KBA-Rückrufe, technische Normen (UN R13, UN R90, ECE, DIN, ISO), Trailer-Modelle und Erkenntnisse aus der Marktanalyse.

REST API

Den Knowledge Graph in APON einbinden.

BASE_URL/api/

Lesen Endpunkte

GET /api/stats
Gesamtstatistiken des Graphen — Knoten, Kanten, Version
curl BASE_URL/api/stats
GET /api/bauteile?q=suchbegriff
Bauteile suchen nach Teilenummer, Name oder Typ. Optional: &hersteller=BPW &baugruppe=BG_BREMSSCHEIBE
curl "BASE_URL/api/bauteile?q=bremsscheibe&hersteller=BPW"
GET /api/cross-references/{teilenummer}
Kompatible Teile anderer Hersteller für eine Teilenummer (Cross-References). Optional: &hersteller=SAF
curl "BASE_URL/api/cross-references/05.397.28.01.0"
// → 9 kompatible Alternativen (Knorr-Bremse, Hendrickson, SAF ...)
GET /api/zulieferer
Alle Tier-1 und Tier-2 Zulieferer. Optional: ?kategorie=Lager
curl "BASE_URL/api/zulieferer?kategorie=Lager"
// → SKF, Schaeffler, Timken
GET /api/reklamationen
KBA-Rückrufe und Reklamationen. Optional: ?hersteller=Krone
curl "BASE_URL/api/reklamationen?hersteller=Krone"

Integration

Der Server lädt den Graphen (silver_lake_merged.json, v11.6) beim Start in den RAM. Alle Abfragen laufen in Echtzeit gegen den In-Memory-Index — keine Datenbank nötig.

Python:

import requests

# Cross-References für ein BPW-Teil
r = requests.get("BASE_URL/api/cross-references/05.397.28.01.0")
data = r.json()
for ref in data["cross_references"]:
    print(f"{ref['teilenummer']} ({ref['hersteller']})")

JavaScript:

const resp = await fetch("/api/chat", {
  method: "POST",
  headers: { "Content-Type": "application/json" },
  body: JSON.stringify({ message: "Zulieferer für Bremsen?" })
});
const { answer, tools_used } = await resp.json();
